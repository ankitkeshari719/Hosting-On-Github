Hosting an app on GitHub

GitHub is a great place for hosting your HTML websites. It’s free and you can even use your own domain names.

This blog let you know a step by step way to host your site on github.

What is GitHub?

GitHub is Git’s cloud-based publishing tool and hosting platform. It also has a desktop application for locally storing projects. It was developed by Chris Wanstrath, PJ Hyett, Tom Preston-Werner and Scott Chacon using Ruby on Rails, and started in February 2008. The company, GitHub, Inc., has existed since 2007 and is located in San Francisco.

Git is used by programmers, developers, and designers to store projects and keep track of changes to their files. It basically is a version control system like any other like CVS and Subversion.

Now, what makes git different from other version control systems is Git is a distributed system . In addition to connecting to a central repository developers can connect to another developers repository. This allows teams to share code without affecting the central repository.

Steps for Hosting a Website on GitHub

  • Create a GitHub account on GitHub site.

  • Download Git depending on your operating system (Window/Mac/Linux). Install it and open the app (Git UI) then log in using the account you just created.

  • Go to GitHub site, login using your github credencial and click on New button to create a new repository.

  • Now, open your folder into git bash on your desktop.

  • Initialise git using git init command.

  • Use git status command to check the status of all files.

  • Add your files to master (default branch provided by git) branch using git add <Filename> command.

    Note: You can add all file using git add . command.

  • Commit your changes using git commit -m "message"

  • Go to GitHub site and open your created repository. Now, you have to clone your repository remote url.

  • Add your remote url in your application using git remote add origin <Remote url that you just copied. > command.

  • Push your changes to your repository using git push -u origin master command.

  • Create a new branch gh-pages in repository using git checkout -u gh-pages command.

    Note: Branch name must be gh-pages.

  • Add, commit and push all your application files to gh-pages branch.

    • git status
    • git add .
    • git commit -m "Your message"
    • git push origin gh-pages

  • Now, switch back to master branch using git checkout master command.

  • Merge gh-pages branch to master using git merge gh-pages command.

  • Againg, push your changes to your repository using git push -u origin master command.

  • Your website is ready, check your-username.github.io/<repository_name>.
